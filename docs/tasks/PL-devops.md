# Production List: Девопсер

## Must

- Подготовить backend-сервер на Ubuntu.
- Установить Node.js 20 LTS.
- Настроить `.env` для backend.
- Настроить SQLite-файл и путь `DB_PATH`.
- Запустить backend через PM2.
- Настроить Nginx reverse proxy.
- Подключить HTTPS через Let's Encrypt.
- Проверить `/api/health`.
- Подготовить Vercel-проект для frontend.
- Задать `VITE_API_URL` с публичным HTTPS API.
- Проверить frontend build.
- Проверить регистрацию и логин на проде.
- Проверить открытие игры с телефона.
- Проверить время сервера и расписание события.
- Подготовить быстрый план перезапуска в день события.
- Проверить CORS между frontend и backend.
- Проверить, что backend не использует временный `JWT_SECRET`.
- Проверить права на SQLite-файл.
- Проверить, что PM2 поднимает backend после рестарта.
- Проверить Nginx-конфиг после изменения.
- Проверить HTTPS-сертификат.
- Проверить доступность API из мобильной сети.
- Подготовить резервную копию пустой БД перед событием.

## Nice

- Настроить PM2 logs для быстрой диагностики.
- Добавить простой health-check в заметки запуска.
- Подготовить команды остановки и перезапуска backend.
- Проверить свободное место на сервере.
- Проверить нагрузку при нескольких одновременных пользователях.
- Подготовить короткую инструкцию отката frontend build.
- Подготовить копирование БД после события.
- Проверить прод на двух мобильных браузерах.
- Проверить, что Vercel использует свежие env-переменные.
- Подготовить список контактов и доступов на день запуска.

## Defer

- Повторная проверка HTTPS ближе к событию.
- Повторная проверка времени сервера в день запуска.
- Ручное сохранение копии БД после завершения события.
- Дополнительная проверка мобильного доступа из офисной сети.
- Уточнение команд перезапуска после финального деплоя.
