# Оценка ролей

Ключевое ограничение проекта: 4 дня, 0 рублей, один человек на подготовку и запуск. Поэтому роли ниже описывают не отдельных людей, а необходимые зоны ответственности.

## Роли

| Роль | Решение | Обоснование |
|---|---|---|
| Разработчик | Да | Нужен для всех игровых систем: авторизации, состояния, сбора предметов, инвентаря, дарения, раскрытия, личной тройки, коллективной цели и финала. |
| Куратор подарочного контента | Да | Нужен для каталога подарков: подобрать изображения, названия и описания, выдержать тёплый праздничный тон и убрать неловкий юмор, стереотипы и военную риторику. |
| Художник / Дизайнер UI | Да | Нужен для мобильного интерфейса, праздничной сцены, карточек подарков, визуального прогресса, анимации раскрытия и финального экрана. |
| Девопсер | Да, минимально | Нужен для подготовки окружения, запуска фронтенда и бэкенда, базы данных, переменных окружения и быстрой проверки перед событием. |
| Геймдизайнер | Да, после запуска | В активной разработке отдельная роль не нужна: GDD уже задаёт решения. После плейтеста геймдизайнер нужен как диагностик результатов: сопоставить поведение игроков с целевыми динамиками и найти причины проблем. |
| Тестировщик | Да, минимально | Нужен быстрый внутренний плейтест основных сценариев. Из-за срока полноценная отдельная роль тестирования не выделяется. |
| Композитор / Саунд-дизайнер | Нет | Звук не является обязательным слоем презентации из-за офисного контекста. |
| Нарративный дизайнер / Сценарист | Нет | Сюжетной структуры нет; короткие тексты входят в контент и интерфейс. |
| Аналитик данных | Нет | Проект одноразовый, без долгосрочной продуктовой аналитики. Нужны только базовые результаты плейтеста. |

## Объединённые роли

- Разработчик + тестировщик → Разработчик — проверка в основном техническая и сценарная, времени на отдельный цикл тестирования нет.
- Девопсер + мониторинг события → Девопсер — масштаб маленький, нужен только запуск и быстрая диагностика.
- Куратор подарочного контента + короткие тексты → Куратор подарочного контента — подарки, сообщения и финальные формулировки должны звучать едино.
- Художник / Дизайнер UI + визуальная проверка UX → Художник / Дизайнер UI — визуальные ассеты и интерфейс работают как одна презентационная система.

## Итоговые роли

1. Разработчик.
2. Куратор подарочного контента.
3. Художник / Дизайнер UI.
4. Девопсер.
