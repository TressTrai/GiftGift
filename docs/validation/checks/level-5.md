# Проверка уровня 5: Механическая спецификация

## 1. Внутренняя проверка

*Не противоречат ли друг другу компоненты уровня?*

### Матрица конфликтов

| Системы | Что может пойти не так | Разрешение |
|---|---|---|
| Сцена и сбор предметов + Инвентарь | Два игрока почти одновременно собирают один предмет. | Предмет получает тот, чей запрос обработан первым; второй игрок получает сообщение, что предмет уже собран. Правило закреплено в системе сцены. |
| Сцена и сбор предметов: активный сбор + периодическая выдача | Периодическая выдача может сделать активный сбор на сцене ненужным. | Периодическая выдача остаётся вспомогательной частью системы сцены: она поддерживает изобилие для пассивных игроков, а активный сбор даёт дополнительные предметы. |
| Сцена и сбор предметов + Переполнение сцены | Новые предметы появляются, когда все точки заняты. | Новые предметы добавляются только на свободные точки; если свободных точек нет, сцена не переполняется. |
| Дарение и трансформация + Получение подарков | Получатель офлайн в момент дарения. | Подарок сохраняется как нераскрытый и будет виден при следующем заходе. |
| Дарение и трансформация + Получение подарков | Получатель онлайн в момент дарения. | Подарок создаётся так же, как для офлайн-получателя; способ уведомления передаётся на уровень 6. |
| Дарение и трансформация + Коллективная цель | Непонятно, когда растёт коллективная цель: при сборе или при дарении. | Коллективная цель растёт только при дарении, а не при сборе предмета. |
| Инвентарь + Личная тройка целей | Подарок засчитался в личную тройку, а затем игрок передарил его дальше. | Засчитывание не сбрасывается; подарок остаётся доступным для дальнейшего дарения. |
| Коллективная цель + Личная тройка целей | Один подарок может одновременно влиять на личную и коллективную цель. | Это допустимо: коллективная цель растёт от факта дарения, личная тройка — от получения нужного подарка. |
| Личная тройка целей + Competition (соревнование) | Личная цель может превратиться в личный счёт или рейтинг. | Тройка видна только самому игроку; засчитываются только подарки от других, а не скорость или количество относительно других игроков. |
| Дарение и трансформация + Анонимность | Полная анонимизация может ослабить социальную связь. | Анонимность является выбором при каждом дарении, а не обязательным режимом по умолчанию. |
| Авторизация и профиль + Дарение | У двух игроков одинаковые имена. | Имена могут совпадать; запись игрока остаётся уникальной. Различение одинаковых имён требует аккуратного представления на уровне 6. |
| Авторизация и профиль + Дарение | Игрок пытается подарить самому себе. | Свой профиль исключается из списка получателей. |

### Граничные случаи

| Ситуация | Что происходит | Разрешение |
|---|---|---|
| Первый игрок заходит в начале дня | Получать подарки пока не от кого. | Игрок может собирать предметы и дарить; периодические предметы поддерживают запас для будущего дарения. |
| Игрок не заходит после регистрации | Другие могут выбрать его получателем. | Нераскрытые подарки накапливаются и будут доступны при следующем заходе. |
| Очень низкая активность | Коллективная цель может стать недостижимой. | Adaptive (адаптивная) цель пересчитывается в контрольные моменты и финал всё равно оформляется позитивно. |
| Очень высокая активность | Игроки быстро заполняют коллективную цель. | Прогресс достигает завершённого состояния, но финал остаётся в 19:00; дарить можно продолжать. |
| Все точки сцены заняты | Новым предметам некуда появиться. | Новые предметы не добавляются до появления свободной точки. |
| Много нераскрытых подарков | Игрок может накопить большой список. | Нераскрытые подарки сохраняются; игрок раскрывает их в любом порядке. |
| Нет подарков из личной тройки | Личная цель долго не закрывается. | Тройка остаётся активной; игрок может просить коллег вживую и продолжать дарить ненужные подарки другим. |
| Финал наступил, игрок офлайн | Игрок не видит кульминацию синхронно. | Финал происходит для активных игроков; результат может быть доступен для просмотра позже, если это поддержит уровень 6. |
| Игрок забыл пароль | Игрок не может войти в свой профиль. | Для небольшого корпоративного события допустима ручная помощь организатора; автоматическое восстановление не является обязательной механикой. |

## 2. Нисходящая проверка

*Не противоречат ли решения этого уровня решениям уровней 0-4?*

**Соответствует ли сложность систем ограничениям уровня 0?** Да.

- Системы используют простые действия: собрать, открыть, подарить, посмотреть прогресс.
- Самая сложная часть, адаптивная коллективная цель, опирается на одну метрику: количество дарений.
- Языковая модель не участвует во время игры, поэтому механики не зависят от внешнего сервиса.

**Покрывают ли системы основной цикл и пики уровня 2?** Да.

- Сбор предметов и периодическая выдача поддерживают получение от механики.
- Дарение и получение подарков поддерживают Gift-Giving Delight (радость дарения).
- Коллективная цель поддерживает главный пик дня.
- Личная тройка целей добавляет вторичный мотив живого взаимодействия.

**Соответствуют ли сцены структуре времени уровня 3?** Да.

- Каждая сцена поддерживает короткий заход на 2-5 минут.
- Основная сцена, инвентарь, дарение и раскрытие можно проходить независимо и останавливать после завершённого действия.
- Финал привязан к 19:00.

**Соответствует ли парадигма систем уровню 4?** Да.

- Коллективная цель использует Adaptive (адаптивную) парадигму.
- Все остальные системы используют Consistency (консистентную) парадигму.
- Скрытая адаптация коллективной цели не меняет фиксированные правила действий игрока.

**Поддерживают ли глобальные законы эстетики уровня 1?** Да.

- Запрет подарка самому себе поддерживает социальную направленность.
- Скрытие точных чисел коллективной цели помогает избежать Competition (соревнование).
- Изобилие и отсутствие штрафов поддерживают лёгкий праздничный тон.

## Найденные проблемы

Пробел, выявленный при проверке: периодическая выдача предметов должна быть явно описана внутри системы сцены, потому что она поддерживает изобилие для пассивных игроков. Пробел устранён: периодическая выдача добавлена в файл сцены как подсистема.
