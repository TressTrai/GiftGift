# Проверка уровня 0: Контекст и ограничения

## 1. Внутренняя проверка

*Не противоречат ли друг другу компоненты уровня?*

**Соответствует ли модальность взаимодействия платформе?** Да.

- Платформа задана как мобильный браузер.
- Основной ввод через касания соответствует мобильному контексту.
- Основной вывод через экран телефона соответствует коротким заходам и игре по ссылке.
- Звук указан как дополнительная обратная связь, поэтому игра не зависит от условий, в которых звук может быть выключен.
- Портретная ориентация, действия одной рукой и отказ от сложных жестов согласуются с мобильным фокусом.

**Образуют ли заявленные ограничения совместимую рамку проекта?** Да.

- Срок 4 дня, команда из 1 человека и бюджет 0 рублей задают жёсткую рамку, но не конфликтуют с выбранным форматом лёгкой веб-игры.
- Корпоративная аудитория 20-30 человек согласуется с архетипом party game (игра для компании), адаптированным под асинхронный формат.
- Косметический уровень интеграции темы снижает нагрузку на разработку: праздник задаёт тон и контент, но не требует сложной нарративной системы.
- Использование языковой модели только для подготовки статического контента согласуется с ограничениями срока и бюджета и не вводит зависимость игровой логики от внешнего сервиса.

## 2. Нисходящая проверка
