# Interface Contract: Куратор подарочного контента ↔ Разработчик

## Разработчик ожидает от куратора подарочного контента

- Файл `public/assets/catalog.json`.
- Ровно 50 записей каталога.
- Уникальные `id` в формате `gift-N`.
- Поле `imageKey`, совпадающее с `id`.
- Название `name` длиной 1-4 слова.
- Описание `description` длиной примерно 50-120 символов.
- Отсутствие обидного, слишком личного, гендерно стереотипного и непонятного контента.

Формат записи:

```json
{
  "id": "gift-1",
  "name": "Термокружка",
  "description": "Хранит кофе тёплым до обеда. И настроение — тоже.",
  "imageKey": "gift-1"
}
```

Срок: до финального подключения каталога и изображений.

## Куратор подарочного контента ожидает от разработчика

- Подтверждение финальной структуры `catalog.json`.
- Проверку, что все `id` и `imageKey` корректно загружаются.
- Сообщение об отсутствующих или лишних записях.
- Сохранение решения: каталог существует только на клиенте, сервер не хранит названия и описания.

Срок: до финальной вычитки каталога.
