# Level 6: Presentation Architecture

**Проект:** Февромарт — Игра Дарения Подарков  
**Дата:** 2 марта 2026  
**Depends on:** L0-context.md через L5-mechanics.md

---

## 1. Adjudication (Как определяются и сообщаются результаты)

**Transparency:** Прозрачная и мгновенная

- Все основные действия просты и всегда успешны (дарение, раскрытие, сбор предметов)
- Нет fail states кроме race condition при сборе предметов (предмет уже собран другим)
- Коллективная цель визуально понятна (прогресс = визуальное заполнение пространства)

**Timing:** Мгновенная

- Сбор предмета: тап → предмет исчезает со сцены → появляется в инвентаре (мгновенно)
- Дарение: подтверждение → подарок отправлен → сразу видишь визуальный результат (если добавился элемент на сцену)
- Раскрытие: тап → анимация 1.5-2.5 сек → результат

**Edge case handling:**

- Race condition (предмет уже собран): предмет исчезает + текст "Уже собран!" над местом где был предмет

**Rationale:**

- Служит Gift-Giving Delight: мгновенная реакция на действия поддерживает момент сюрприза и удовлетворение от дарения
- Соответствует Casual player profile: нет сложных вычислений или неясных результатов
- Fellowship: визуальный результат коллективных усилий сразу виден всем

## 2. Information Management (Что игроки знают и когда)

**Что всегда видимо:**

- Личная тройка целей (на главном экране в разделе Инвентарь)
- Коллективный прогресс (визуальное заполнение сцены)
- Свой инвентарь (предметы + подарки)
- Список всех коллег для дарения

**Что скрыто до раскрытия:**

- Содержимое нераскрытого подарка (полная тайна — универсальная упаковка)
- От кого подарок (если анонимный)
- Сообщение от дарителя (если есть)

**Что скрыто всегда:**

- Цели других игроков (нужно спрашивать вживую)
- Статус онлайн/офлайн коллег
- Точная цифра коллективной цели (игроки видят только визуальный прогресс, нет "347/800 подарков")

**Как раскрывается:**

- Уведомление о новых подарках: при заходе в игру (не real-time)
- Содержимое подарка: при раскрытии (анимация 1.5-2.5 сек)
- Превью трансформации: при выборе подарка для дарения (ДО подтверждения)

**Delivery pattern:** Push при заходе

- При открытии игры: "У тебя 3 новых подарка!" (если есть)
- Между заходами: нет уведомлений (подарки накапливаются пассивно)

**Rationale:**

- Полная тайна до раскрытия максимизирует Gift-Giving Delight (момент сюрприза)
- Приватные цели мотивируют живое общение (Fellowship через реальное взаимодействие)
- Скрытая точная цифра коллективной цели поддерживает магию shared triumph (кажется что "мы вместе создали", а не "система подстроила")
- Casual player: информация pushed (не нужно искать), но не overwhelming

## 3. Content Delivery (Как контент подаётся игрокам)

**Pattern:** Hybrid — фронтлоадный статический контент + респавн механика

**Статический контент (подготовлен заранее):**

- Каталог подарков: 40-60 уникальных (изображение + описание каждого)
- Визуальная сцена: 1 большая иллюстрация праздничного пространства

**Динамическая выдача:**

- Предметы на сцене: респавн каждые 5-10 минут, 3-5 предметов на свободные точки (8-12 точек всего)
- Подарки от коллег: асинхронно, когда кто-то дарит
- Личная тройка целей: 3 случайных подарка из каталога, обновляется при закрытии тройки

**Pacing of reveal:**

- Первый заход: ~8-12 предметов доступны на сцене сразу
- Каждый последующий заход (через 1-3 часа): новые предметы появились
- За день: игрок увидит ~50-55 подарков (сам собрал + получил от коллег)

**Gating:** Нет жёсткого gating

- Весь каталог подарков доступен с первого дня (случайная выдача)
- Нет unlocks или прогрессивного открытия контента

**Rationale:**

- Casual player + короткие сессии: контент должен быть сразу доступен, без препятствий
- 40-60 уникальных при ~50 за день = каждый встретится 1-2 раза (разнообразие + узнавание)
- Респавн создаёт изобилие (нет дефицита, легко дарить) — служит Gift-Giving Delight
- Статический контент = 4 дня разработки реализуемо (LLM генерация до релиза)

## 4. State Persistence (Что игра запоминает)

**Что персистентно в течение игрового дня:**
- Инвентарь игрока (все предметы + подарки)
- Нераскрытые подарки (не исчезают между заходами)
- Личная тройка целей + прогресс (прогресс сохраняется между сессиями)
- История дарений (кто кому что подарил, когда, анонимно/открыто)
- Коллективный прогресс (глобальный счётчик подарков)
- Предметы на сцене (глобальное состояние для всех игроков)

**Что НЕ персистентно:**
- UI состояния (на каком экране был, прокрутка)
- Время последнего захода (не отслеживается для показа статуса онлайн/офлайн)

**Как коммуницируется:**
- При заходе: синхронизация с сервером → "У тебя X новых подарков!" (если есть)
- Автосохранение: каждое действие сразу сохраняется на сервере
- Возврат после перерыва: состояние полностью восстанавливается (инвентарь, цели, прогресс)

**After игровой день (после 19:00):**
- Сцена завершена: финальная визуализация доступна для просмотра
- Инвентарь сохранен

**Rationale:**
- Total persistence служит Fellowship: все вклады сохраняются, коллективный результат накапливается
- Casual player: не нужно помнить что было — всё сохранено, легко вернуться
- Асинхронный multiplayer: состояние персистентно пока другие игроки офлайн
- Одноразовая игра (1 день): нет meta-progression между днями

## 5. Feedback (Как игра сообщает что происходит)

**Timing:** Immediate (мгновенная реакция на действия)

**Intensity:** Punchy для ключевых моментов, subtle для фоновых событий

**Modality:** Визуальная + аудио (звуковые эффекты, без фоновой музыки)

**Ключевые feedback моменты:**

**1. Раскрытие подарка (КРИТИЧНО для Gift-Giving Delight):**

- Анимация: 1.5-2.5 сек (shake/wiggle упаковки → burst/reveal с частицами → fade-in картинки подарка)
- Звук: сюрприз/reveal звук (приятный, не резкий)
- Визуал: большая картинка подарка + описание + от кого (если не анонимно) + сообщение (если есть)

**2. Сбор предмета со сцены:**

- Визуал: предмет исчезает с лёгкой анимацией (fade-out или particle effect)
- Звук: короткий "собрано" звук
- UI feedback: предмет появляется в инвентаре (можно показать микро-уведомление "+1 предмет")

**3. Дарение подарка:**

- Визуал: "Подарок отправлен! Коллега будет рад" (текст)
- Если попал на boundary коллективного прогресса (5-10%): анимация добавления элемента на сцену + "Ты помог украсить праздник!"
- Если не попал: "Подарок отправлен! Коллективный прогресс: ~45%" (примерный %)
- Звук: короткий успех звук

**4. Закрытие тройки целей:**

- Визуал: конфетти/частицы на 1-2 секунды + "Тройка собрана! Отличная работа"
- Звук: победный звук (короткий, праздничный)
- Затем: плавное появление новой тройки целей

**5. Race condition (предмет уже собран):**

- Визуал: предмет исчезает + текст "Уже собран!" (всплывающий над местом)
- Звук: нейтральный или без звука (не негативный)

**6. Финал в 19:00:**

- Полноэкранная celebration анимация (fireworks, заполненная сцена)
- Текст: "Мы вместе украсили праздник!"
- Звук: кульминационный праздничный звук
- Длительность: 3-5 секунд, затем можно вернуться смотреть результат

**Emotional register:** Игривый и тёплый (без emoji)

- Примеры текстов: "Подарок отправлен! Коллега будет рад", "У тебя 3 новых подарка! Интересно, что там?", "Уже собран! Кто-то опередил"

**Звуковое оформление:**

- Только звуковые эффекты (5-10 звуков из бесплатных библиотек)
- Без фоновой музыки
- Можно отключить в настройках

**Rationale:**

- Immediate feedback служит Gift-Giving Delight: мгновенное подтверждение действий усиливает момент сюрприза
- Punchy на ключевых моментах (раскрытие, закрытие тройки) создаёт эмоциональные пики
- Wonder aesthetic: визуальные эффекты (частицы, анимации) делают моменты приятными
- Тёплый тон поддерживает Fellowship: не нейтрально, но и не навязчиво
- Звуки без музыки: управляемый scope (4 дня) + не мешает в офисе

## 6. Cognitive Framing (Как игроки учатся и понимают)

**Onboarding approach:** Краткий визуальный туториал (1-2 экрана)

**Структура обучения:**

**При первом запуске игры:**

**Экран 1 (Core Loop):**

- Визуал: Скриншот сцены с тапом на предмет
- Текст: "Собирай предметы на сцене"
- Визуал: Скриншот раскрытия подарка
- Текст: "Раскрывай подарки от коллег"
- Визуал: Скриншот экрана дарения
- Текст: "Дари другим"
- Визуал: Заполняющаяся сцена
- Текст: "Вместе украшаем праздник!"

**Экран 2 (Детали):**

- Визуал: Тройка целей
- Текст: "Собери эти 3 подарка! Проси коллег подарить тебе нужное"
- Визуал: Превью трансформации
- Текст: "Подарки трансформируются при передаривании — превью показывается перед дарением"

**Навигация:** Свайп между экранами, кнопка "Начать играть" на втором экране

**После туториала:** Можно вызвать из настроек ("Как играть" или "Помощь")

**Progressive disclosure:** Minimal

- Вся механика объяснена за 2 экрана (~20-30 секунд)
- Никакой дополнительной информации в процессе игры (интуитивный UI)
- Все действия понятны из визуала (кнопки, карточки, иконки)

**UI hierarchy (что подчёркнуто):**

- Главный экран = Сцена (первое что видишь)
- Тройка целей постоянно видна в Инвентаре
- Badge на разделе "Инвентарь" если есть нераскрытые подарки (привлекает внимание)
- Коллективный прогресс визуально заметен на сцене (заполнение пространства)

**Mental model support:**

- Tooltips: нет (всё объяснено в туториале)
- Help system: повторный просмотр туториала из настроек
- In-game hints: нет (интуитивный UI не требует)

**Метафора:** "Офисный праздник — собираем и дарим подарки, вместе украшаем пространство"

- Знакомый контекст: подарки, празднование
- Простая механика: собирать, раскрывать, дарить

**Rationale:**

- Casual player profile: должны понять за 30-60 секунд — 2 экрана с визуалом это обеспечивают
- Минимальная cognitive load: не перегружаем текстом, показываем скриншоты
- Интуитивный UI: после туториала не нужны дополнительные подсказки
- Можно вернуться: если забыл, открываешь помощь из настроек
- 4 дня разработки: простой туториал = быстрая реализация

## 7. UI/UX Specification

**Platform considerations:**

- Мобильный фокус (touch интерфейс)
- Вертикальная ориентация
- Короткие сессии (2-5 минут)

**Navigation structure: Tab bar (2 основных раздела + профиль)**

**Tab 1: Сцена** (главный экран, открывается по умолчанию)

- Большая иллюстрация праздничного пространства (зимне-весенняя палитра: холодные голубые/белые + тёплые жёлтые/розовые)
- Предметы для сбора (тапабельные, анимированные/светящиеся для отличия от декора)
- Элементы коллективного прогресса (статичные, декоративные, добавляются каждые 5-10%)
- Pan/zoom по сцене
- 8-12 фиксированных точек для респавна предметов

**Tab 2: Инвентарь**

- Вертикальный scroll, три секции:
    1. **Личная тройка целей** (верх, всегда видима):
        - 3 карточки подарков горизонтально
        - Прогресс каждой цели (галочка ✓ если собран)
    2. **Новые подарки** (если есть):
        - Badge "У тебя 3 новых подарка! Интересно, что там?"
        - Список нераскрытых подарков (универсальная упаковка)
    3. **Мой инвентарь** (все раскрытые):
        - Предметы (серая/нейтральная рамка)
        - Подарки (золотая/праздничная рамка + gift icon в углу)
        - Grid layout, можно скроллить

**Tab 3: Профиль**

- Имя игрока
- Кнопка "Изменить имя"
- Кнопка "Изменить пароль"
- Кнопка "Как играть" (повторный просмотр туториала)
- Переключатель "Звуки вкл/выкл"

**Дополнительные экраны (открываются из контекста):**

**Детальный просмотр предмета/подарка:**

- Большая картинка
- Описание (1-2 предложения)
- Если подарок: от кого (или "Анонимный подарок"), когда получен
- Если предмет: когда собран со сцены
- Кнопка "Подарить это"

**Экран дарения:**

- Выбор получателя: список всех игроков (случайный порядок при каждом открытии, алфавитный внутри списка)
- Превью: карточка текущего подарка + текст "→ Превратится в: [Название]" + маленькая иконка (если передаривание)
- Текстовое поле для сообщения (опционально, 100-150 символов макс, счётчик)
- Галочка "Анонимно?"
- Кнопка "Подарить"

**Раскрытие подарка:**

- Полноэкранная анимация (1.5-2.5 сек): shake/wiggle → burst/reveal
- Результат: картинка + описание + от кого + сообщение (если есть)
- Автоматически добавляется в инвентарь после просмотра

**Финал в 19:00:**

- Полноэкранная celebration (fireworks, заполненная сцена)
- Текст "Мы вместе украсили праздник!"
- Длительность 3-5 сек, затем возврат к сцене

**Visual style:**

- Палитра: холодные голубые/белые + тёплые жёлтые/розовые (зимне-весенний переход)
- Стиль: приятные иллюстрации, не перегруженные деталями
- Подарки: визуально разнообразные и красивые (Wonder aesthetic)
- UI элементы: мягкие rounded углы, приятные тени
- Шрифты: читаемые на мобильном, дружелюбные (не строгие)

**Tone of voice:** Игривый и тёплый, без emoji

- "Подарок отправлен! Коллега будет рад"
- "У тебя 3 новых подарка! Интересно, что там?"
- "Тройка собрана! Отличная работа"
- "Уже собран! Кто-то опередил"
- "Мы вместе украсили праздник!"

**Rationale:**

- 2 основных раздела (Сцена + Инвентарь) = минимальная cognitive load для Casual player
- Главный экран = Сцена (визуально приятное, сразу понятно что делать) — служит Wonder + Gift-Giving Delight
- Тройка целей постоянно видна → мотивация спрашивать коллег → Fellowship
- Случайный порядок получателей → равномерное распределение внимания → Fellowship
- Зимне-весенняя палитра → метафора объединения, визуально гармонично

## 8. Cognitive Budget Final Check

**Total presentation complexity:** Low-Medium (соответствует Casual player profile)

**Cognitive load breakdown:**

**Tracking load:**

- Личная тройка целей (3 подарка, видима постоянно)
- Нераскрытые подарки (badge показывает количество)
- Коллективный прогресс (визуальный, не требует запоминания цифр)
- **Итого: ~3-5 элементов для отслеживания** (в пределах Casual бюджета 3±2)

**Decision load:**

- Что собрать со сцены? (любой предмет, нет плохих выборов)
- Что подарить? (выбор из инвентаря, превью показывает результат)
- Кому подарить? (список всех, случайный порядок снижает paralysis)
- Анонимно или открыто? (бинарный выбор)
- **Итого: простые решения, небольшое количество опций, нет необратимых последствий**

**Memory load:**

- Между сессиями: ничего не нужно помнить (всё сохранено)
- Правила игры: объяснены за 2 экрана, интуитивны (собирать/дарить/раскрывать)
- Цели коллег: можно спросить заново (не критично помнить)
- **Итого: минимальная memory load**

**Execution load:**

- Простые tap действия
- Pan/zoom на сцене (стандартный mobile gesture)
- Scroll в списках
- **Итого: нет сложных комбинаций, всё standard mobile interactions**

**Reduction strategies applied:**

- **Simplification:** Убрали статус онлайн/офлайн (не нужная информация)
- **Automation:** Коллективная цель адаптируется автоматически (игрок не думает о балансе)
- **Visualization:** Прогресс показан визуально, не цифрами (легче воспринимать)
- **Offloading:** Превью трансформации показывается системой (не нужно запоминать правила)
- **Push information:** Уведомления о подарках pushed при заходе (не нужно искать)

**Coherence with Level 1:**

- ✅ Casual player profile: cognitive load соответствует (3-5 trackables, простые решения, минимальная память)
- ✅ Gift-Giving Delight aesthetic: presentation усиливает (анимация раскрытия, тёплый тон, визуальная приятность)
- ✅ Fellowship aesthetic: presentation поддерживает (приватные цели → живое общение, случайный порядок → равномерное внимание)
- ✅ Wonder aesthetic: визуальная приятность (палитра, иллюстрации, анимации)
- ✅ Session structure (2-5 мин): UI не перегружен, действия быстрые

**No audience shift:** Presentation architecture соответствует изначальному Casual player profile.
