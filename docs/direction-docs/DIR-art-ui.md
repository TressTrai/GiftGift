# Direction Document: Художник / Дизайнер UI

## Назначение роли

Художник / дизайнер UI отвечает за визуальную читаемость мобильной игры, праздничную сцену, подарки, интерфейсные элементы, визуальный прогресс и модальные состояния.

Визуальный стиль должен поддерживать Gift-Giving Delight (радость дарения), Wonder (восхищение) и Fellowship (товарищество), не превращая игру в соревновательную витрину.

## Общий стиль

- Мягкая зимне-весенняя атмосфера.
- Тёплые оттенки: кремовый, персиковый, нежно-жёлтый, пыльно-розовый, светло-голубой.
- Праздничность без перегруза.
- Предметы и UI должны быть читаемыми на телефоне.
- Визуальные элементы не должны требовать долгого рассматривания.

## Подарки

Каждый подарок изображается как отдельный предмет, а не сцена и не персонаж.

Требования:

- PNG;
- 512×512 px;
- прозрачный фон;
- предмет занимает 70-80% кадра;
- предмет расположен по центру;
- небольшие отступы по краям;
- плавные линии и лёгкие тени или объём;
- без мелких деталей, которые теряются на мобильном.

Каждый подарок должен быть визуально уникален и не путаться с соседними.

## Сцена

Основная сцена должна работать как пространство поиска и лёгкого удивления.

Важные элементы:

- фон `2160×4800`;
- интерактивная область с pan/zoom;
- 12 точек появления предметов;
- элементы коллективного прогресса;
- праздничная, но не шумная композиция.

## UI

UI строится вокруг быстрого мобильного использования:

- нижняя навигация: сцена, инвентарь, профиль;
- badge для новых подарков;
- крупные touch-зоны;
- модальные окна без визуальной тяжести;
- ясные состояния: подарок раскрыт, подарок не раскрыт, предмет можно подарить.

## Анимации и эффекты

Анимации должны поддерживать радость открытия, но не задерживать игрока дольше необходимого.

Ключевые эффекты:

- сбор предмета;
- отправка подарка;
- раскрытие подарка;
- завершение личной тройки;
- финальный экран.

## Ограничения

- Не использовать визуальные элементы лидерборда.
- Не показывать точное число коллективной цели.
- Не делать подарки гендерно разделёнными.
- Не перегружать сцену мелкими деталями.
